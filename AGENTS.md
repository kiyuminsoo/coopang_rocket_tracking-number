# Codex 작업 지침

## 언어 규칙
- 모든 답변, 계획, 분석, 확인 질문, 작업 완료 보고는 한국어로 작성한다.
- 사용자가 영어를 요청하지 않는 한 영어로 답하지 않는다.
- 파일명, 폴더명, 명령어, 코드, 함수명, 라이브러리명, 에러 원문은 원문 그대로 유지한다.
- 영어 에러 메시지는 원문을 보여준 뒤, 바로 아래에 한국어로 쉽게 설명한다.

## 작업 방식
- 코드를 수정하기 전에 어떤 파일을 왜 수정할지 한국어로 먼저 설명한다.
- 수정 후에는 변경한 파일과 변경 이유를 한국어로 요약한다.
- 사용자가 명확히 승인하지 않은 큰 구조 변경은 하지 않는다.
- 배포나 Git push 전에는 변경사항을 한국어로 요약하고 확인을 요청한다.
